Rstring

Description

Rstrings are inconsistent due to their hacky implementation.
This means that you can access indeces or sclices as you would a list or iterate through it with a for each loop.

Like HTML or python, feel free to use either single or double quotes to define a string. This means you do not have to escape one if you use the other. Example: Cstring contraction = "That's pretty cool";
Cstring quote = 'and then he said, "Hello, how are you?"'
The Char type does not exist in MCL

Implementation

Rstrings in MCL are very dependent on the work done by Ts with string parsing into char arrays.
Because of this, every Rstring is either a char[] or a native string.

Declaration

Cstring var = "a";
Cstring var = 'a';
Cstring var = "Hello, world!";
Cstring var = 'Hello, world!';

Operators

//assignment operators
var = 'a'

Substrings

Identical to Python.
