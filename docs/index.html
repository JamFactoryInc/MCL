Minecraft Command Language

Welcome to MCL

What is MCL?

MCL is is a language designed to be intuitive, consistent, and concise.
Most of it is a hybrid of C# and Python with a few extra sprinkles here and there.

MCL is compiled into a standard Datapack. No mods, no nonsense.

MCL allows the user to use a high-level language to write Datapacks instead of wrangling the normal commands in a txt file.

Runtime variables

You heard me. I've spent a long time allowing as much runtime functionality as absolutely possible.
Summoning 100 Zombies named "Zombie 0" to "Zombie 100" is as easy as for i in 100
{
Zombie z = Zombie();
z.name = "Zombie " + i;
}

The "It just works" mentality is dumb

While giving the user the highest level of abstraction away from the actual commands, MCL retains the ability to inject commands inline for those hands-on vets among you, as well as having a clear distinction between runtime and compiletime values.

Trident

Trident is a language that was developed by Energyxxer for the same purpose as MCL.
It's a Datapack language.
I've tried it and it's not for me. The reason I didn't want to learn Trident was its complexity and its lack of runtime variables.
Everything else about it is great, and I recommend giving it a look
