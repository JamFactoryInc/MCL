Minecraft Command Language

Welcome to MCL

What is MCL?

MCL is a datapack generator program that reads python code
No mods, no new languages or syntax, no nonsense.

MCL allows the user to use python to define complex behavior that is difficult to manage with obtuse datapack syntax.

Runtime over Compile Time

MCL was built to allow for as much runtime functionality as possible.
If you're unclear, runtime is undefined behavior. For example: user_input = input() print(user_input) There's no way to make this work unless you define the variable user_input at runtime.
On the other hand, compile-time is pre-defined behavior. Such as: print(1+2) This will always print 3, and so there is no need to ever add 1 and 2. All that is compiled is the equivalent of print(3)

Why Choose MCL

There are reasons to and not to use MCL. Here are some pros and cons:

Pros

  • MCL is loosely-typed and supports many data types
  • MCL uses identical syntax to python (with some necessary amendments)
  • Performance aside, essentially anything possible in vanilla python is possible in MCL

Cons

  • MCL is not as efficient. Many nbt operations are done in order to allow for runtime capability and these are more expensive
  • MCL is abstracted away from the commands. While command injection/direct manipulation is supported and encouraged, the resulting datapack will be essentially nonsense and hard to edit yourself if you wish to make changes outside of the MCL coding env
  • MCL is young and only maintained by one developer. Some day support and updates may stop if I become too busy or am otherwise unable
